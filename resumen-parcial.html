Historia y Características de Java

La Plataforma Java

📅 Cronología

  • Proyecto Green — James Gosling crea Oak para electrónica de consumo (microondas, TV)
  • Star Seven (*7) — prototipo de controlador del hogar con pantalla táctil
  • 1994 — FirstPerson cierra, mercado TV no despegó
  • Bill Joy ve el potencial para Internet → renombra a Java

🔄 Arquitectura: Write Once, Run Anywhere

  • .java → javac → .class (bytecode)
  • Bytecodes = código neutro, independiente del SO
  • Intérprete: lee instrucción por instrucción (flexible, lento)
  • JIT (Just In Time): compila bytecode a código nativo → velocidad
  • Lema: "Write Once, Run Anywhere"

🛠️ Herramientas del SDK

  • javac — compilador (.java → .class)
  • java — ejecuta programas
  • javadoc — genera documentación HTML
  • jar — empaqueta archivos en .jar
  • jdb — depurador (debugger)

📦 Ediciones de Java

  • J2SE — escritorio y uso general (Standard Edition)
  • J2EE — aplicaciones empresariales / servidores
  • J2ME — dispositivos móviles y embebidos (Micro)

IDEs gratuitos: Eclipse, NetBeans

IDEs pagos: Borland JBuilder, Visual Age

ℹ️
C/C++ compilan a código máquina específico (hay que recompilar por SO). Java compila a bytecode portable → la JVM lo ejecuta en cualquier plataforma.

  • Widening (automático): int → double

⬆️ Jerarquía de Operadores (alta → baja prioridad)

1️⃣ () paréntesis 2️⃣ ! negación 3️⃣ * / % 4️⃣ + - 5️⃣ < > <= >= 6️⃣ == != 7️⃣ && 8️⃣ || 9️⃣ = asignación

⚠️ Errores comunes

  • if (x = 5) — usa ==, no =
  • 5 + 3 * 2 = 11, no 16 (por precedencia)
  • 10 / 4.0 = 2.5 (double, no int)
  • División por cero → ArithmeticException

💡 Cortocircuito

  • &&: si el primer operando es false, no evalúa el segundo
  • ||: si el primer operando es true, no evalúa el segundo
  • Usar && en lugar de & (bit) para mayor eficiencia

Sección 4

Estructuras Condicionales & Switch

if-then

if (condicion) { // se ejecuta solo si es true }

if-else

if (condicion) { // true } else { // false }

else-if (cadena)

if (c1) { // c1 true } else if (c2) { // c2 true } else { // ninguna }

⚡ Para en la primera condición verdadera

Switch

Sintaxis

switch (variable) { case 1: // acción break; case 2: // acción break; default: // si no hay match }

Reglas clave

  • break — sale del switch; sin él los casos siguen en cascada (fall-through)
  • default — se ejecuta cuando ningún case coincide
  • Ideal para valores discretos exactos (no rangos)
  • Funciona con int, char, String, enum
⚠️
Error común: if (edad >= 18 && edad <= 65); — el punto y coma cierra el if y el bloque se ejecuta siempre.

Sección 5

Bucles

while — pre-prueba

int i = 0; while (i < 5) { System.out.println(i); i++; }
  • Evalúa condición antes de ejecutar
  • Puede no ejecutarse nunca

do-while — post-prueba

do { System.out.println(i); i++; } while (i < 5); // ← ;
  • Ejecuta cuerpo al menos una vez
  • Ideal para menús interactivos
  • Requiere ; al final

for — rango conocido

for (int i = 0; i < 5; i++) { // ↑inicialización ↑condición ↑incremento System.out.println(i); }
Inicialización: se ejecuta UNA sola vez Condición: se evalúa antes de cada iteración Incremento: se ejecuta después de cada iteración

🧮 For-each (enhanced for)

for (int n : arreglo) { System.out.println(n); }

⚠️ n es una copia, modificarla no cambia el arreglo original

⚠️ Bucle infinito

// Sin incremento → bucle infinito while (i < 10) { // falta i++ } // i-- con condición i<10 → infinito for (i = 0; i < 10; i--) {}

🔍 Bucles anidados

for (int i=0; i<f; i++) { for (int j=0; j<c; j++) { // cada celda de matriz } }

Variables de control deben tener nombres distintos

Bucle¿Cuándo usarlo?¿Mínimo de ejecuciones?
forSe conoce el número de iteraciones de antemano0
whileSe repite mientras se cumpla una condición (rango indefinido)0
do-whileMenús, input que debe ejecutarse al menos una vez1

Sección 6

Arreglos

Arreglo 1D (Vector)

// Declarar y crear int[] nums = new int[5]; // Inicializar directamente int[] nums = {10, 20, 30}; // Acceso: índices 0 a length-1 nums[0] = 10; // primer elemento nums.length; // sin paréntesis

Arreglo 2D (Matriz)

// Declarar y crear int[][] m = new int[3][4]; // Inicializar directamente int[][] m = {{1,2},{3,4}}; // Acceso [fila][columna] m[1][0] = 3; // fila 1, col 0

📌 Reglas fundamentales

  • Índices: 0 hasta length - 1
  • arreglo.length — atributo, sin ()
  • Valores por defecto: int[] → 0, boolean[] → false, objetos → null
  • Acceder a índice inválido → ArrayIndexOutOfBoundsException

🔁 Recorrer con for

for (int i=0; i < nums.length; i++){ System.out.println(nums[i]); } // Recorrer matriz 2D for (int i=0; i<m.length; i++) for (int j=0; j<m[i].length; j++) System.out.print(m[i][j]);

⚠️ Errores comunes

  • Índice fuera de rango: new int[5] → válidos 0-4
  • Bucle empezando en 1 ignora el índice 0
  • Llamar método sobre null → NullPointerException
  • Matriz irregular: filas con distinta longitud provocan excepciones
💡
Vector = arreglo unidimensional (1D) · Matriz = arreglo bidimensional (2D)
⚙️
Sección 7

Métodos en Java

Anatomía de un método

public static double larger(double x, double y) { // ↑modificadores ↑tipo retorno ↑nombre ↑parámetros formales return x > y ? x : y; // cuerpo del método }
Modificadores: public static Tipo retorno: double Nombre: larger Firma = nombre + parámetros (SIN tipo retorno)

5 Propiedades de un método (D.S. Malik)

#Propiedad¿Dónde aparece?
1Nombre del métodoEncabezado
2Número de parámetrosEncabezado
3Tipo de dato de cada parámetroEncabezado
4Tipo de retorno (tipo del método)Encabezado
5El código (cuerpo del método)Cuerpo — solo métodos definidos por el usuario

🔄 Métodos con retorno

  • Declaran un tipo (int, double, etc.)
  • Requieren return valor;
  • Se usan dentro de expresiones
  • double r = larger(5, 10);

🚫 Métodos void

  • Palabra clave: void
  • Realizan una tarea, NO devuelven nada
  • Se invocan como sentencia independiente
  • NO se pueden usar en expresiones
  • Pueden usar return; para salida anticipada

⚡ Flujo de ejecución

  • Siempre empieza en la primera instrucción de main
  • Los demás métodos se ejecutan solo cuando son invocados
  • Al terminar un método, el control vuelve a la instrucción inmediatamente después de la invocación

Parámetros formales vs. actuales

Tipo¿Qué es?Ejemplo
Formal Variable declarada en el encabezado del método double x, double y en la definición
Actual Valor o variable listado en la invocación larger(5.6, 10.8) → 5.6 y 10.8
❌
En la invocación NO se escribe el tipo: larger(int one, 29) → ILEGAL. Solo los valores: larger(one, 29)

Sección 8

Sentencia return & Scope

✅ Correcto

if (x > y) return x; else return y; // Método termina aquí

⚠️ Código inalcanzable

if (x > y) return x; else return y; // ↓ NUNCA se ejecuta → error compilación System.out.println("hola");
⚠️
Reglas del return: Termina el método inmediatamente. El tipo debe coincidir con el declarado. Si el compilador detecta una ruta sin return → error "missing return statement".
💡
Los métodos void pueden usar return; (sin valor) para salir antes del final del cuerpo.

Reglas de Scope (Alcance)

ReglaDetalle
Identificador localVisible solo desde su punto de declaración hasta el fin de su bloque (})
Java: sin métodos anidadosNO se puede definir un método dentro de otro método

No re-declarar en bloque internoUna variable del bloque externo del método no puede re-declararse en un bloque interno
Identificador static fuera de métodosAccesible desde cualquier método static, a menos que sea ocultado por una variable local del mismo nombre

Sección 9

Parámetros y Paso por Valor

ConceptoDescripciónEjemplo
Parámetros formales Los de la definición del método double x, double y
Parámetros actuales Los valores al invocar el método larger(5.6, 10.8)
Regla del protocolo Deben coincidir en: 1) cantidad, 2) orden y 3) tipo de dato

Primitivos (int, double, char, boolean)

  • Se pasa una COPIA del valor
  • Cambios dentro del método NO afectan el original
  • "La fotocopia"

Variables de referencia (objetos)

  • Se copia la DIRECCIÓN del objeto
  • Ambos apuntan al mismo objeto
  • Cambios al objeto SÍ son visibles fuera

Ejemplo paso por valor (primitivos)

public static void main(String[] args) { int number = 6; cambiar(number); // pasa COPIA de 6 System.out.println(number); // imprime 6 (sin cambio) } public static void cambiar(int num) { num = 15; // solo cambia la copia local, no el original }

String — INMUTABLE

  • Al asignar con =, crea un NUEVO objeto
  • El parámetro formal apunta al nuevo objeto
  • El parámetro actual (original) no cambia
  • Comportamiento parecido al paso por valor
void f(String pStr) { pStr = "Dia soleado"; // nueva referencia, str intacto }

StringBuffer — MUTABLE

  • Modifica el mismo objeto en memoria
  • Los cambios SÍ se ven desde el llamador
  • Usar append() y delete()
  • Inicializar siempre con new StringBuffer(...)
void f(StringBuffer pStr) { pStr.append(" que tal"); // modifica el objeto original }
📌
Las variables de referencia como parámetros son útiles cuando necesitás: devolver más de un valor, cambiar el objeto del llamador, o ahorrar memoria no copiando datos grandes.

Sección 10

Sobrecarga de Métodos (Overloading)

Mismo nombre, diferente firma

public static int larger(int x, int y) { ... } public static double larger(double x, double y) { ... } public static char larger(char a, char b) { ... } // Firma = nombre + lista de parámetros (tipos y orden) // El tipo de retorno NO es parte de la firma

¿Cuándo son diferentes las listas de parámetros formales?

  • Diferente número de parámetros, O
  • Mismo número pero diferente tipo en al menos una posición

Selección en la invocación

El compilador elige la versión sobrecargada según los parámetros actuales:

larger(5, 3) // → larger(int, int) larger(5.0, 3.0) // → larger(double, double) larger('A', '9') // → larger(char, char)
❌
Cambiar solo el tipo de retorno NO es sobrecarga válida → Error de compilación.
Misma firma (nombre + params) con distinto tipo de retorno = error.

Sección 13

Layout Managers

🌊 FlowLayout

  • Izquierda → derecha, luego baja de línea
  • Default de JPanel
  • Ideal para botones y campos simples

🧭 BorderLayout

  • 5 regiones: NORTH, SOUTH, EAST, WEST, CENTER
  • Default de JFrame
  • Solo un componente por región

⊞ GridLayout

  • Cuadrícula de filas × columnas
  • Todas las celdas del mismo tamaño
  • new GridLayout(3, 2)

JLabel con Imágenes (NetBeans)

Cargar y mostrar imagen en JLabel

// La imagen debe estar en el mismo paquete que el .java ImageIcon icono = new ImageIcon(getClass().getResource("foto.png")); miLabel.setIcon(icono);

Redimensionar imagen

1
Crear ImageIcon con getClass().getResource("foto.png")
2
Obtener Image: icono.getImage()
3
Escalar: imagen.getScaledInstance(ancho, alto, Image.SCALE_SMOOTH)
4
Crear nuevo ImageIcon con la imagen escalada
5
Asignar con miLabel.setIcon(nuevoIcono)
ImageIcon icono = new ImageIcon(getClass().getResource("foto.png")); Image img = icono.getImage(); Image scaled = img.getScaledInstance(150, 150, Image.SCALE_SMOOTH); miLabel.setIcon(new ImageIcon(scaled));
⚠️
Ruta absoluta = MAL: new ImageIcon("C:/mis fotos/foto.png") solo funciona en tu PC.
Ruta relativa = BIEN: getClass().getResource("foto.png") funciona en cualquier equipo.
